SCORM 2004 3rd Edition Sample Run-Time Environment Version 1.0.2 Readme
Sample Run-Time Environment

Introduction

Welcome to the Sharable Content Object Reference Model (SCORM) 2004 3rd Edition Sample Run-Time Environment (RTE) Version 1.0.2 (Referred to hereafter as Sample RTE). The Sample RTE is intended to provide an example implementation of the concepts described in the SCORM. Specifically, it was developed in order to provide an example of the SCORM Run-Time Environment (Application Programming Interface (API) and Data Model) and SCORM Sequencing and Navigation as they might be applied in a Learning Management System (LMS) environment. Robust exception and error handling has not been included in this sample code, nor have performance optimizations been considered.

Even though there are a number of valid Web-enabled SCORM implementations, this version of the Sample RTE was implemented as a Web-based client/server application using HTML, JavaScript, JavaServer Pages (JSP), Java Applets and Java Servlets (Refer to the "Design Information" section of this Readme for additional Sample RTE design information).

Installation

Prerequisites

It is recommended that you read through all of the Installation Instructions prior to beginning installation.

This release of the Sample RTE requires the Java Development Kit (JDK) from any one of the following Java versions:

  • Java SE 6, any update level
  • J2SE 5.0, Update 10 or later
  • J2SE 1.4.2_13 or later
The latest version of this software is available for download free of charge at http://java.sun.com/javase/downloads/index.jsp. Archived versions are available at http://java.sun.com/products/archive/. Refer to this site for a list of specific Java versions verified.

Notes:

  • The Sample RTE requires the full Java Development Kit. The Java Runtime Environment (JRE) alone is NOT sufficient for this software.
  • The Sample RTE client may be accessed from a machine other than the one on which it is installed using Microsoft® Internet Explorer®. In this case, the Java Runtime Environment must be installed on the second machine.

After installing the Java Development Kit, you must do the following:

  1. Create an environment variable named "JAVA_HOME" and set its value to the root directory of the Java Development Kit.
    • For Windows XP: Right click on "My Computer", select "Properties", select the "Advanced" tab and then click on "Environment Variables". Add a JAVA_HOME entry in the User or System Variables that points to the location where the Java Development Kit was installed (e.g., C:\j2sdk1.4.2_05).
    • For Windows Vista: Open the “Start" menu and right click on "Computer", select "Properties", select the "Advanced System Settings" link on the left navigation menu, and then click on "Environment Variables". Add a JAVA_HOME entry in the User or System Variables that points to the location where the Java Development Kit was installed (e.g., C:\j2sdk1.4.2_05).

  2. Add the full path of the Java Development Kit bin directory (where the java.exe is located) to the PATH environment variable.
    • For Windows XP: Right click on "My Computer", select "Properties", select the "Advanced" tab and then click on "Environment Variables". Add the Java Development Kit bin directory to the Path by adding a semicolon followed by a reference to the JAVA_HOME environment variable followed by "\bin" (i.e., %JAVA_HOME%\bin).
    • For Windows Vista: Open the “Start” menu and right click on "Computer", select "Properties", select the "Advanced System Settings" link on the left navigation menu, and then click on "Environment Variables". Add the Java Development Kit bin directory to the Path by adding a semicolon followed by a reference to the JAVA_HOME environment variable followed by "\bin" (i.e., %JAVA_HOME%\bin).

Constraints

As of this release, the Sample RTE has been tested on Microsoft® Windows® XP Professional (SP2), and Vista all on the Intel® Pentium® PC platform. Testing was also performed with Microsoft Internet Explorer 6.0 and Internet Explorer 7.0.

The Sample RTE has not been formally tested with any other browsers on any other platforms than those indicated above.

Installation

Installation of the Sample RTE is to be done only after installing the Java Development Kit as described in the Prerequisites.

The Sample RTE is available for download as a self-extracting executable install file. Once you have completed downloading the file, perform the following steps:

  1. Double-click on the downloaded self-extracting executable file to begin the installation process.
  2. Click the "Next" button to continue from the "Welcome" page.
  3. Read the "License Agreement"
    • Click the "Yes" button to agree and continue with the install.
    • Click the "No" button if you do not agree with the terms of the license and wish to exit the install.
  4. Choose whether you would like the Sample RTE to be installed for Anyone who uses this computer (all users) or Only me.
  5. The recommended root installation directory is "C:\ADL". You may choose another root installation directory, but the Sample RTE install will always append the "\ADL\SCORM_2004_3rd_Ed_Sample_RTE_V1.0.2" sub-directory to the specified root installation directory. The remainder of this Readme document assumes that the Sample RTE will be installed to "C:\ADL\SCORM_2004_3rd_Ed_Sample_RTE_V1.0.2". This is referred to as "<drive>:\<install_directory>". If you override the default to use a different root installation directory, please take that into consideration when reading the remainder of this Readme.
  6. Click the "Next" button
  7. Click the "Install" button to begin the installation
  8. You will have the option to view a copy of this Readme document.
    • Check the "Show the Readme file" checkbox to view the Readme.
    • Uncheck the "Show the Readme file" checkbox to skip viewing the Readme.
  9. Click on the "Finish" button to complete the installation.
Note: The installation will not remove/un-install previous versions of the Sample RTE.

Note: The apache-tomcat-5.5.20 Servlet Engine and Web server will be installed and pre-configured to bind to port 8080.

Configuration

      2. Click on "Tools" -> "Internet Options"

      4. Ensure that the "Every visit to the page" radio button is selected

      4. Ensure that the "Every visit to the page" radio button is selected
      5. Click "OK" and then "OK" again.

    • For J2SE 1.4.2_XX

      2. Double click on the "Java Plug-in" icon
      3. Select the "Cache" tab
      4. Un-check the "Enable Caching" option

  • Uninstall the Sample RTE using the Windows Control Panel.
    • For Windows XP, the Windows Control Panel "Add/Remove Programs" feature can be used.
      1. Open the Windows "Control Panel"
      2. Double-Click on "Add or Remove Programs" icon
      3. Highlight the "ADL SCORM 2004 3rd Ed. Sample Run-Time Environment Version 1.0.2"
      4. Click "Remove" and follow the instructions

These methods of uninstalling will not remove files or directories that have been altered in the course of running the Sample RTE. In order to complete the process of uninstalling the Sample RTE, the following steps will need to be completed:
  1. Delete the directory that the Sample RTE was installed to (<drive>:\<Install Directory>\SCORM_2004_3rd_Ed_Sample_RTE_V1.0.2 by default) along with any files or directories contained within it
  2. Delete the <drive>:\SCORM3EDSampleRTE102Files directory

Running the Sample RTE

Instructions

The instructions provided are for running the Sample RTE on a Windows platform. This is the only platform that the software has been tested on and is the only platform for which instructions are provided at this time.

To Run the Sample RTE:

First, start the apache-tomcat-5.5.20 servlet container and Web server that is provided with this installation. To do this:

  1. Click on the shortcut at Start Menu -> All Programs -> ADL -> SCORM_2004_3rd_Edition_Sample_Run-Time_Environment_version_1.0.2 -> Start Server. You will see a command prompt window appear and show several lines of textual information. This window will remain open until you stop the server.
  2. Next click on the Start Menu -> All Programs -> ADL -> SCORM_2004_3rd_Edition_Sample_Run-Time_Environment_version_1.0.2 -> SCORM 2004 3rd Edition Sample Run-Time Environment version 1.0.2. This will launch the Sample RTE Main page. Log in with the username: "admin", password: "admin". (Be sure that Microsoft Internet Explorer 6.0 or greater is your default browser.)

Online help pages are available once the Sample RTE is started. "Help" links appear at the bottom of each page.

To close the Sample RTE, first close the browser in which it is running. To stop the apache-tomcat-5.5.20 servlet container and Web server, click on the Start Menu -> All Programs -> ADL -> SCORM_2004_3rd_Edition_Sample_Run-Time_Environment_version_1.0.2 -> Stop Server.

To return the Sample RTE to a "clean" state, log into the Sample RTE and from the main menu choose the Clear Database option. This will remove all course files that have been imported as well as any information associated with them, including IMS SSP buckets and related information; and will remove all user registration information. All Global Objective information will also be removed.

Course Navigation User Interface

In addition to any User Interface (UI) controls provided within course content, the Sample RTE provides several mechanisms for navigating through a course that are conformant to Section 2.1.7 of the SCORM 2004 3rd Edition Conformance Requirements Version 1.0. These mechanisms include the navigation buttons in the top frame of the Sample RTE, such as the Quit, Suspend, Previous, and Continue buttons, and the Table of Contents provided in the left-hand frame.

Navigation Buttons

The navigation buttons provided by the Sample RTE in the top frame will be visible when they are not explicitly “hidden” by an activity via the hideLMSUI element within the manifest. When an activity explicitly hides any of the buttons, they will not appear in the top frame.

The following describes the behavior of the navigation buttons when they are not hidden:

  • Quit – This button will issue an Exit All navigation request to the Sample RTE’s Sequencer; the Sample RTE will end a user’s current attempt on a course. The user’s progress and course status information for the current attempt will no longer be available if the user makes another attempt on the course.
  • Suspend – This button will issue a Suspend All navigation request to the Sample RTE’s Sequencer; the Sample RTE will end a user’s current attempt on a course and will also ‘bookmark’ the user’s position within the course. The user’s progress and course status information will be persisted. When the user starts the course again, they will resume the course from the same content they triggered the suspend.
  • Previous – This button will issue a Previous navigation request to the Sample RTE’s Sequencer, resulting in launching the activity identified prior to the current activity in the course’s activity tree.
  • Continue – This button will issue a Continue navigation request to the Sample RTE’s Sequencer, resulting in launching the activity identified after the the current activity in the course’s activity tree.

Table of Contents

A Table of Contents (TOC) will be provided by the Sample RTE when a course includes Choice equal to “true” anywhere within its activity tree. When a course includes Choice equal to “false” at every level of its activity tree, the table of contents will not appear.

The following describes the items in the TOC when Choice is set to “true” in the course:

  • Selectable – Items that are selectable will be indicated with black text and the cursor will change as the user mouses over them.
  • Not Selectable – Items that are not selectable will be indicated by gray text or black text and the cursor that does not change when the user mouses over them.

Design Information

The Sample RTE Version 1.0.2 is composed of several distinct and separate components. These components are as follows:

  1. Sample Run-Time Environment Server Component: Implemented using Java Applications and Java Servlets.
  2. Run-Time Environment Client Component: Implemented using JSP, Java Applets, HTML and JavaScript.


Sample Run-Time Environment Server Component

The Sample RTE Server Component is implemented using Java Servlets and Java Applications. The Servlets respond to requests from the Run-time Environment Client Component and are responsible for Data Model persistence as well as sequencing and parsing logic. The source files for the server component are located beneath the Sample_RTE\source\src directory. In addition, the JavaDoc associated with the source files is located at Sample_RTE\api\index.html. The modules that make up the Server Component are listed below:

  • ADL Validator: This module is responsible for validating the imsmanifest.xml file that is required in SCORM Content Aggregation Content Packages. The ADL Validator is also responsible for parsing the XML file and returning a DOM tree representation to the Sample RTE Server Component. Refer to the JavaDoc Package, "org.adl.validator" for details.
  • Utilities: This module contains general purpose utility Java files. Refer to the JavaDoc Package, "org.adl.util" for details.
  • ADL Sequencer: This module is responsible for handling sequencing based on the IMS Simple Sequencing Specification and SCORM 2004 3rd Edition Sequencing and Navigation Version 1.0. It contains classes that create an initialized activity tree, maintain the activity tree throughout the sequencing process and handle all sequencing logic. Refer to the JavaDoc Package, "org.adl.sequencer" for details.
  • Sample RTE: The Sample RTE Package is the hub of all of the Server Component packages. It contains the logic to utilize the other packages when needed. It also contains Java classes that interact with Java Server Pages to implement server side functionality. Refer to the JavaDoc Packages,"org.adl.samplerte.server" and "org.adl.samplerte.util" for details.
  • Data Models: This package contains the classes that maintain the SCORM Run-Time Environment Data Model and the Navigation Data Model. Refer to the JavaDoc Package "org.adl.datamodels" for details.
  • Shareable State Persistence: This package contains the classes used to provide the IMS Shareable State Persistence implementation. Refer to the JavaDoc Package "org.ims.ssp.samplerte.server" for details.

Sample Run-Time Environment Client Component (Java Files)

The Sample RTE Client Component consists of a sample user interface implemented in HTML and JavaScript and the Sample RTE API Instance implemented as a Java Applet. The Applet is downloaded to the client when the user accesses the Sample RTE page through a Web browser. Sample Run-Time Environment Client Component (Web Files)
The Sample RTE Client Component is comprised of the following modules which can be found beneath the installation location in the Sample_RTE\source\SampleRTE\LMSClient directory:

  • Administrative: This module contains files that change passwords, delete courses, manage global objectives, create or delete users, delete courses and clear the Sample RTE database. The files that make up the Administrative module are located in the "admin" directory.
  • Help: This module contains information pages on how to use the Sample RTE. The files that make up the Help module are located in the "help" directory.
  • Import: This module is responsible for managing the course import process. These files utilize the Sample RTE Server side Java Component to handle parsing, validation and database access. The files that make up the Import module are located in the "import" directory.
  • Includes: The Sample RTE utilizes a style sheet (*.css) that handles the interface look and feel. See sampleRTE_style.css for details. This file is located in the "includes" directory.
  • Run-Time: This module contains the files used to deliver a SCORM Conformant Content Aggregation Content Package. In addition, the SCORM API is included. All images used in the Sample RTE menu are located in the "menu-images" directory located in this module. The files that make up the Run-Time module are located in the "runtime" directory.
  • Special State: This module contains information pages that describe special sequencing states. It is possible that content will not be delivered as result of a sequencing request. In that event these pages inform the user about what state the sequencer is in. Note: These pages are included to help debug sequencing in content. There is no requirement by the SCORM that they be presented. LMSs may handle sequencing states in their own manner. The files that make up the Special State module are located in the "specialstate" directory.

Building the Sample RTE

Note: It is not a requirement to build the Sample RTE before using it. It will install as a fully operational unit. It is only necessary to build the Sample RTE if the source code has been changed.

In order to compile the Sample RTE, the full installation instructions must be completed first. The Sample RTE build file depends on a full apache-tomcat-5.5.20 install, along with resources provided in the installation of the Sample RTE. It is important to note that when the Sample RTE is built, it overwrites the previously installed Sample RTE. To move back to the original Sample RTE, the un-install must be executed and then the downloaded self-extracting executable must be re-run.

The Sample RTE is built using Apache Ant, a Java-based build tool. Before building the Sample RTE, Apache Ant must be installed. For information about and downloads of Ant, refer to http://ant.apache.org/.

To build the Sample RTE:
  • Open a command prompt and navigate to the directory where the Sample RTE is installed (Default: C:\ADL\SCORM_2004_3rd_Ed_Sample_RTE_V1.0.2)
  • From this point, navigate to the Sample_RTE\source directory
  • At the prompt type "ant".

The XML Ant build files will compile all Java class and jar files needed by the Sample RTE. In addition the files will be installed to apache-tomcat. (Note: For changes to take effect, the tomcat server will need to be restarted). This completes the compilation of the Sample RTE.

Source File Locations

  • All JSP, HTML, and JavaScript files are located at <drive>:\<install_directory>\Sample_RTE\source\SampleRTE\LMSClient.
  • The Sample RTE Java files are located in their respective package locations. For example the data model classes are located at "org\adl\datamodels". The root of the Java source directory is <drive>:\<install_directory>\Sample_RTE\source\src. Here you will find the build script and the root "org" directory.

Known Issues

  1. If a package manifest contains references to Sharable Content Objects (SCOs) that are located at an external Web site, a cross-domain JavaScript security error will occur. This may prevent the SCOs from communicating with the Sample RTE provided API.


  2. The Internet Explorer browser must be set to check for a new page every visit to the page. To configure the browser:
    1. Open a Microsoft Internet Explorer Browser
    2. Click on "Tools" -> "Internet Options"
    3. On the "General" tab, "temporary internet files" sub-section, click "Settings"
    4. Ensure that the "Every visit to the page" radio button is selected
    5. Click "OK" and then "OK" again.

  3. It is possible that switching between different versions of the Sample RTE could cause an Applet Error if Applet Caching is enabled. If this occurs, the following steps can be taken to correct the issue:
    • For J2SE 1.4.2_XX
      1. Open the Windows "Control Panel"
      2. Double click on the "Java Plug-in" icon
      3. Click on the "Cache" tab
      4. Click the "Clear" button in the upper right-hand corner
      5. Click Yes when prompted to confirm
      6. Uncheck the Enable Caching check box in the upper left-hand corner
      7. Click the Apply button.

    • For J2SE 5.0
      1. Open the Windows "Control Panel"
      2. Double click on the "Java Plug-in" icon
      3. Click on the "General" tab at the top of the dialog box
      4. Under the "Temporary Internet Files" section, click the "Settings" button
      5. Click the "View Applets" button
      6. Ensure that the check box labeled "Enable Caching" is unchecked (for the ADL applets).

    • For Java SE 6
      1. Open the Windows "Control Panel"
      2. Double click on the "Java" icon
      3. Click on the "General" tab at the top of the dialog box
      4. Under the "Temporary Internet Files" section, click the "Settings" button
      5. Ensure that the check box labeled "Keep temporary files on my computer" is unchecked.

Implementation Decisions
Case Sensitive URLs

  • URLs that are located in the imsmanifest.xml file and that identify launch locations and other hrefs are case sensitive in the Sample Run-Time Environment. For example, sco.html is different from SCO.HTML and from Sco.html. If the case used to specify a file name does not match the case of the actual file, an error will result while running the course.


Maximum Allowable Course Size

  • The maximum allowable size for a course imported into the Sample Run-Time Environment is approximately 2.0 gigabytes. Courses exceeding 2.0 gigabytes may not import correctly.


Manifest Encodings

  • Any course imported into the Sample Run-Time Environment which contains foreign characters in the manifest, must have the imsmanifest.xml file encoded in either UTF-8 or UTF-16 (Big Endian or Little Endian).


Supported Java Versions
  • The following list describes Java Development Kit versions that have been tested with the Sample RTE as of the date of this release:
    • Java SE 6, any update level
    • J2SE 5.0, Update 10 or later
    • J2SE 1.4.2_13 or later
